# Taniti — travel site (static)

A small multi-page HTML/CSS/JS site for the fictional island of Taniti.

## GitHub Pages (hosted URL)

After you push this repo to GitHub:

1. On GitHub, open the repository → **Settings** → **Pages** (under “Code and automation”).
2. Under **Build and deployment**, set **Source** to **Deploy from a branch**.
3. Choose **Branch: `main`** and folder **`/ (root)`**, then **Save**.
4. Your site will be available at:

   **`https://<your-username>.github.io/<repository-name>/`**

   Example: if your username is `hanna` and the repo is `taniti-travel`, the URL is  
   `https://hanna.github.io/taniti-travel/`

5. Wait one or two minutes for the first build, then refresh. Use **Actions** or the Pages settings banner if the URL does not appear right away.

Relative paths (`css/`, `js/`, page links) work correctly under the `/<repo>/` base path.

## Local preview

Open `index.html` in a browser, or serve the folder with any static server.
